## 0. 평가 환경의 결정적 제약 (Executive Summary)

본 평가는 EVAL-002 본래 사양인 **검색 API(`POST /api/v1/search`) 호출 기반 종합 평가**를 수행하지 못했다. 환경 점검 결과 다음 두 가지 결정적 제약이 확인되었기 때문이다.

### 제약 1. 임베딩 0건 — 벡터 검색·Re-ranker 작동 불가
```sql
SELECT COUNT(*) AS total, COUNT(embedding) AS embedded FROM regulations;
 total | embedded
-------+----------
   127 |        0
```
- `regulations` 테이블에 127개 청크가 적재되어 있으나, **`embedding` 컬럼은 모두 NULL**.
- 원인: `.env` 파일 미생성, `UPSTAGE_API_KEY` 미주입 → `EmbeddingPipeline.run()` 미실행 (PHASE1_PROGRESS.md 35행에 "운영 환경 실측 실행 대기 중"으로 기록됨).
- 결과: `PostgresVectorSearchAdapter` 호출 시 결과 0건. `HybridSearchAdapter`는 BM25 결과만으로 동작하고, `UpstageSolarRerankerAdapter`는 단건/0건 입력으로 무력화됨.

### 제약 2. 코퍼스 핵심 본문 미적재 — 정답 조항의 100%가 DB에 부재
| 법령 | DB 청크 | 본문 청크 (article != 'N/A') | 골든 정답 조항 |
|---|---:|---:|---:|
| 식품 등의 표시ㆍ광고에 관한 법률 | 1 | **0 (스텁만)** | 제8조 1·2·4호 |
| 건강기능식품에 관한 법률 | 1 | **0 (스텁만)** | 제18조 |
| 화장품법 | 1 | **0 (스텁만)** | 제13조 1·2·3·4호, 제14조 |
| 화장품 표시·광고 관리 지침 | 27 | 0 (전부 `N/A`) | (전체 매칭) |
| 건강기능식품 인체적용시험 표시·광고 가이드라인 | 55 | 0 (전부 `N/A`) | (전체 매칭) |
| 식품등의 부당한 표시 또는 광고의 내용 기준 | 34 | 33 | 제2조 1호/3호/4호 |
| 건강기능식품 표시·광고 자율심의기구 운영규정 | 7 | 7 | (정답 미참조) |
| 온라인 부당광고 사례집 | 1 | 0 | (정답 미참조) |

- 법령 3종(식품표시광고법·건강기능식품법·화장품법)은 `law.go.kr` HTML이 JavaScript 비동기 로드 방식이라 **본문이 1개의 스텁 청크에만 기록**되어 있다. (REGULATIONS_INVENTORY.md 18~19행에 명시된 알려진 제약).
- 골든 세트가 참조하는 unique 정답 조항 17개를 점검한 결과 **17/17(100%)가 DB에 strict 매칭 불가**.
- 식품 부당고시(34건)와 화장품 지침(27건)은 DB에 존재하나, **화장품 지침은 `article_number`가 전부 `N/A`라 조항 단위 매칭이 구조적으로 불가능**하다.

### 평가 방향 (Plan B)
사용자 요청을 가능한 한 충실히 이행하기 위해, 다음과 같이 평가 범위를 좁혀 실행했다.

1. **검색 모드**: BM25 단독 검색 (ParadeDB `chunk_text @@@ ?`, `paradedb.score()` 정렬, `LIMIT 10`). 이는 `HybridSearchAdapter`가 사용하는 `ParadeDbKeywordSearchAdapter`와 동일 SQL이다.
2. **매칭 기준**: 두 가지를 병기한다.
   - **strict**: 법령명 + (조,항,호,목)이 모두 일치 (운영 기준)
   - **loose**: 법령명만 일치 (코퍼스 미적재 진단용)
3. **외부 의존 없음**: API 서버·임베딩 API 호출 불요. PostgreSQL만으로 재현 가능.

따라서 본 보고서의 수치는 **현 시점의 시스템 상한(BM25 단독)** 을 측정한 것이며, 본래 목표였던 "Hybrid + Re-ranker 종합 평가"의 결과가 아니다.

## 1. 종합 성과

| 지표 | 측정값 | 기준 | 판정 |
|---|---:|---:|:---:|
| Recall@5 (strict) | **20.0%** (5/25) | ≥ 80% | ❌ **미달** |
| Recall@10 (strict) | **20.0%** (5/25) | ≥ 90% | ❌ **미달** |
| Recall@5 (loose, 법령명만) | 52.0% (13/25) | (참고) | — |
| Recall@10 (loose, 법령명만) | 52.0% (13/25) | (참고) | — |
| 평균 BM25 후보 수(top10) | 5.6 건 | — | — |
| 평균 검색 latency | 185.6 ms | < 500ms | ✅ 충족 |

**최종 판정: ❌ Phase 1 성공 기준 미달**

추가 사실:
- strict hit 5건의 **전원이 정답 조항이 "전체"(가이드라인 통째 매칭)인 케이스**다. 즉 **세부 조항 단위 strict hit은 0건**이다. 5건은 모두 화장품/객근거 영역의 "화장품 표시·광고 관리 지침"이 정답에 포함된 샘플(GOLDEN-019, 020, 021, 024, 025)이다.
- BM25 검색이 **0건을 반환한 샘플이 5개** 존재한다(GOLDEN-001, 003, 006, 017, 022). 모두 정답이 미적재 법령(식품표시광고법·화장품법·건강기능식품법)뿐인 케이스다.

Recall@5 == Recall@10 인 이유는 BM25가 평균 5.6건만 반환하기 때문이다 (top 6~10에 신규 매칭이 발생하지 않음).

## 2. 위반 유형별 분석

| 유형 | 샘플 수 | Recall@5 (strict) | Recall@10 (strict) | Recall@5 (loose) | Recall@10 (loose) |
|---|---:|---:|---:|---:|---:|
| 질병 예방·치료 표방 | 5 | 0% | 0% | 0% | 0% |
| 의약품 오인 유발 | 5 | 0% | 0% | 60% | 60% |
| 허위·과장 표시 | 5 | 0% | 0% | 100% | 100% |
| 객관적 근거 부재 | 5 | 40% | 40% | 40% | 40% |
| 화장품 효능 광고 위반 | 5 | 60% | 60% | 60% | 60% |
| **전체** | **25** | **20%** | **20%** | **52%** | **52%** |

해석:
- **질병 예방·치료 표방 (0/5)**: 정답이 식품표시광고법·건강기능식품법·식품부당고시에 분포한다. 식품법 두 건은 스텁 청크만 있고, 부당고시 제2조 1호(질병 효능표방 일반조항) 청크는 적재되어 있으나 BM25가 "당뇨병/암/고혈압/혈압 조절" 같은 구체 질병명 쿼리를 *법조문 추상 어휘*("질병의 예방·치료에 효능이 있는 것으로 인식")와 매칭하지 못함. 형태소·동의어 미보강 시 키워드 단독으로는 어휘 격차가 너무 크다.
- **허위·과장 표시 (loose 100%)**: 부당고시 제2조 4호 영역이 DB에 존재하므로 법령명 일치까지는 100% 달성. 그러나 strict는 0% — 조항/호/목 단위 정확 매칭은 BM25가 식별하지 못한다.
- **화장품 효능 광고 위반 (60%)**: 화장품법 본문은 미적재이나, 화장품 표시·광고 관리 지침(27건)에 "아토피, 의사 처방, 천연·유기농" 등 구체 어휘가 포함되어 BM25가 직접 매칭. 단 화장품법 자체의 strict 매칭은 0건.
- **객관적 근거 부재 (40%)**: 화장품 관련 샘플(GOLDEN-019, 020)은 화장품 지침 매칭으로 hit. 식품 관련(GOLDEN-016~018)은 인체적용시험 가이드라인 미적재로 부분 손실.

## 3. 난이도별 분석

| 난이도 | 샘플 수 | Recall@5 (strict) | Recall@10 (strict) | Recall@5 (loose) | Recall@10 (loose) |
|---|---:|---:|---:|---:|---:|
| easy | 10 | 0% | 0% | 30% | 30% |
| medium | 10 | 20% | 20% | 70% | 70% |
| hard | 5 | 60% | 60% | 60% | 60% |
| **전체** | **25** | **20%** | **20%** | **52%** | **52%** |

해석 (역설적 패턴):
- **easy가 가장 낮음(0%)**: easy는 모두 질병·의약품 표방 케이스로, 정답이 식품 3법(스텁) + 부당고시 추상조항이다. 코퍼스 적재 결함이 직격탄으로 작용했다.
- **hard가 가장 높음(60%)**: hard는 5건 모두 화장품 효능 광고 위반인데, 화장품 표시·광고 관리 지침(27건)이 적재되어 있어 정답 일부를 직접 매칭. **검색 알고리즘 자체의 hard 처리 능력 때문이 아니라 코퍼스 적재 편향 때문**으로 해석된다.

즉 이번 난이도별 그래프는 **검색 시스템의 난이도 대응력이 아닌 코퍼스 적재 편향**을 보여주는 지표다.

## 4. 미달 원인 분석

### 원인 1. 코퍼스 적재 미완료 — 1차 책임 (DATA-002 후속 작업 누락)
- 법령 3종(화장품법, 식품표시광고법, 건강기능식품법)이 law.go.kr의 JavaScript 비동기 로드 제약으로 **본문이 추출되지 않은 스텁 청크 1건**으로만 존재.
- REGULATIONS_INVENTORY.md 18~19행에 "P1-2 단계에서 elaw open API 또는 본문 텍스트 추출 예정"으로 명시되었으나 후속 처리가 누락됨.
- 영향: 골든 정답 17개 중 13개가 이 3법에 속함. → strict Recall 상한이 구조적으로 ~24%로 잠김.
- 추가로 화장품 지침/건기식 가이드라인은 적재되었으나 `article_number=N/A`로 통일되어 있어 **조항 단위 strict 매칭이 영구 불가**한 상태다.

### 원인 2. 임베딩 미적재 — 벡터·하이브리드 평가 불가
- 127개 청크 모두 `embedding IS NULL`.
- 원인: 로컬 `.env` 미생성, `UPSTAGE_API_KEY` 미설정. `EmbeddingPipeline.run()` 트리거 미실행.
- 영향: `PostgresVectorSearchAdapter` 응답 0건 → `HybridSearchAdapter`가 사실상 BM25 모드로 다운그레이드, Re-ranker도 무력화. ADR-003/004에서 약속된 임베딩+리랭킹 효과 검증 불가.

### 원인 3. BM25 단독으로는 추상 법조문 ↔ 구체 광고 카피 매칭 곤란
- 적재된 부당고시 청크(제2조 1호 질병 효능표방 일반조항)는 "질병의 예방·치료에 효능이 있는 것으로 인식할 우려가 있는 표시 또는 광고"라는 *추상 표현*이다.
- 골든 쿼리("당뇨병 예방", "고혈압을 잡아드립니다")는 *구체 질병명·구어체*다.
- BM25(기본 한국어 토크나이저)는 어휘 직접 일치가 없으면 매칭 점수를 충분히 부여하지 못함. 5개 샘플은 BM25 결과 0건을 반환했다.
- 이 격차는 임베딩 기반 의미 검색의 본래 역할이다. 즉 벡터 검색 없이 본 도메인에서 BM25만으로 Recall ≥ 80%를 달성하는 것은 비현실적이다.

### 원인 4. 조항 메타데이터 부정확 — strict 매칭 분모를 잠식
- 화장품 지침/건기식 가이드라인 청크는 PDF 파싱 시 `article_number`를 추출하지 못해 일괄 `N/A`로 저장됨.
- 같은 조항이라도 골든 세트는 "제13조 제1항 1호"로, DB는 `N/A`로 라벨되어 strict 비교가 영구 실패.
- 본 평가에서는 정답 조항이 "전체"인 경우만 strict hit으로 계산했으나, 실제 운영에서는 조항 인덱스가 RAG 응답의 인용 신뢰도에 직결된다.

## 5. 개선 방안 (Phase 1 재시도 → Phase 1.5)

### 우선순위 1 — 코퍼스 완전 적재 (가장 큰 효과 기대)
1. **법령 3종 본문 추출**
   - 옵션 A: 국가법령정보센터 OpenAPI(elaw) JSON 응답 사용 → `<조문내용>` 노드 추출
   - 옵션 B: law.go.kr 본문 PDF 다운로드 페이지(`/lsLawListInfo.do`)로 우회 또는 헤드리스 브라우저 렌더링
   - 결과 검증: 화장품법 제13조/제14조, 식품표시광고법 제8조, 건기식법 제18조가 각각 조항 단위 청크로 적재되는지 확인
2. **조항 메타데이터 강화**
   - 화장품 지침·건기식 가이드라인 PDF 청킹 로직 보강: 목차/별표 헤더에서 "□ 화장품법 제13조 제1항 제1호" 같은 인용을 파싱하여 `article_number/paragraph_number/item_number` 컬럼에 백필
   - JSONL 재생성 → `regulations` 재적재 (멱등성 유지)

### 우선순위 2 — 임베딩 적재 후 재평가
1. `.env`에 `UPSTAGE_API_KEY` 주입
2. `./gradlew :api:bootRun` → `POST /api/v1/admin/embedding/pipeline/run`
3. 임베딩 충전 확인: `SELECT COUNT(embedding) FROM regulations;` == 127
4. 본 스크립트(`eval_recall_k.py`)에 벡터 검색 경로 추가 또는 검색 API 호출 모드 추가 → 재실행

### 우선순위 3 — BM25 한국어 토크나이저 보강 (D2 ADR 보강)
- ParadeDB pg_search의 한국어 분석기 설정 점검 (현재 기본값 사용 중)
- 동의어 사전 도입: 광고 어휘("잡아드립니다", "탁월한 효과") → 법령 추상어("예방·치료에 효능") 매핑
- ADR-002 보강 필요

### 우선순위 4 — 골든 세트 조정 (단기 완화책)
- 코퍼스에 본문 부재인 정답 조항을 "전체" 매칭으로 다운그레이드하여 측정 가능 범위 확보 (운영 기준 약화)
- 단 본 조치는 RAG 응답 인용 신뢰도를 검증하지 못하므로, **우선순위 1과 동시 진행하지 않으면 측정만 통과하고 운영 실패**가 발생할 수 있다.

### 우선순위 5 — Re-ranker 실측 평가
- 우선순위 2 완료 후 Hybrid + Reranker 풀스택으로 재평가
- ADR-004의 "Upstage Solar-reranker 채택" 근거(상관·MRR 향상)를 실측 수치로 검증

**판정**: 코드/인프라 측면의 구현은 완료되었으나 데이터 측면(코퍼스 본문 적재, 임베딩 충전)이 미완료 상태이며, 이로 인해 정량 지표가 성공 기준에 크게 미달한다. **Phase 1을 "구현 완료, 평가 미달"로 종료**하고, **Phase 1.5(데이터 보강)** 를 신설하여 우선순위 1~2를 처리한 뒤 본 평가를 재실행해야 한다.
