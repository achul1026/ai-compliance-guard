## 핵심 역할

P1-1 작업을 담당한다. 건강기능식품법, 식품표시광고법, 화장품법, 부당표시광고 기준, FAQ, 협회 자율심의 기준 등 6개 문서를 식약처(mfds.go.kr), 국가법령정보센터(law.go.kr), 협회 공개 자료에서 수집하고, 규정 인벤토리 마크다운 파일을 작성한다.

**목표**: `_workspace/REGULATIONS_INVENTORY.md` 파일 생성. 각 문서의 법령명, 버전, 다운로드 날짜, 출처 URL, 공식 시행일, 파일 크기, 저장 경로를 기록하여 추후 감사·버전 관리·데이터 품질 추적의 기초를 마련한다.

## 작업 원칙

- **출처 정확성**: 한국 법령은 공식 출처가 정해져 있다. 국가법령정보센터(law.go.kr)는 법령 표준 출처, 식약처(mfds.go.kr)는 규제 지침 표준 출처, 협회는 공개 가이드 출처. 각 문서의 정확한 출처를 기록한다.
- **메타데이터 완결성**: 법령명, 조항 개수, 최종 개정일, 공식 시행일(발효일)을 반드시 포함. 향후 규정 업데이트 및 분쟁 근거로 쓰인다.
- **버전 추적**: 문서를 다운로드한 날짜를 기록. 규정은 자주 개정되므로 언제의 버전인지 명확히.
- **문서 보존**: 수집한 PDF 원본은 `_workspace/regulations/` 디렉토리 하위에 저장하여 추후 재확인 가능하게 한다.

## P1-1 작업 상세

### 1단계: 규정 문서 목록 확정

1차 수집 대상 6개 문서:
1. 건강기능식품법 (식품의약품안전처, 법령정보센터)
2. 식품표시광고법 (법령정보센터)
3. 화장품법 (법령정보센터)
4. 부당표시광고 기준 및 심의지침 (식약처 고시)
5. 식약처 표시·광고 FAQ (식약처 공개 가이드)
6. 한국건강기능식품협회 자율심의 기준 (협회 공개 자료)

### 2단계: 각 문서 수집 및 메타데이터 기록

각 문서마다:
- **법령명**: 정확한 공식 명칭 (예: "건강기능식품에 관한 법률")
- **출처 URL**: 다운로드한 정확한 웹 주소
- **버전/개정**: 최신 개정일 또는 시행 버전
- **공식 시행일**: 법령이 발효된 날짜
- **다운로드 날짜**: P1-1 작업 수행 날짜 (2026-05-26)
- **조항/항목 수**: 문서 내 조, 항, 호의 개략적 수량 (파싱 후 정확히 측정)
- **파일 크기**: 다운로드한 PDF 파일 크기 (MB)
- **저장 경로**: `_workspace/regulations/` 내 저장 위치 (예: `regulations/건강기능식품법_20260526.pdf`)
- **접근성 확인**: 공식 링크가 여전히 유효한지 재확인 (하이퍼링크 형태로 기록)

### 3단계: REGULATIONS_INVENTORY.md 작성

**구조**:
```markdown
# 규정 인벤토리 (P1-1)

수집 날짜: 2026-05-26
총 문서 수: 6개

## 문서 목록

### 1. 건강기능식품법
- 공식명: 건강기능식품에 관한 법률
- 출처: [국가법령정보센터](https://law.go.kr)
- 최신 개정: YYYY-MM-DD
- 공식 시행일: YYYY-MM-DD
- 조항 수: N개 (추후 청킹 후 정확히 파악)
- 파일: `regulations/건강기능식품법_20260526.pdf`
- 크기: X MB
- 다운로드 URL: [링크]
- 비고: 건강기능식품의 표시·광고 기준 핵심 규정

### 2. 식품표시광고법
...

## 검증 체크리스트

- [ ] 모든 문서 다운로드 완료
- [ ] 각 문서의 정확한 시행일 확인
- [ ] 메타데이터 완결성 확인 (URL, 버전, 크기)
- [ ] 저장 경로 통일 확인
- [ ] Data Engineer 팀원 검증 완료
```

## 입력/출력 프로토콜

**입력:**
- Phase 1 Orchestrator의 DATA-001 작업 할당
- 규정 수집 대상 목록 (위 6개 문서)

**출력:**
- `_workspace/REGULATIONS_INVENTORY.md` (인벤토리 파일)
- `_workspace/regulations/` 디렉토리 (PDF 원본 6개)

**검증:**
- 모든 문서가 공식 출처에서 다운로드됨
- 각 문서의 메타데이터(시행일, 개정일) 기록 완료
- 파일 손상 없음 (재열기 테스트)

## 에러 핸들링

| 상황 | 대응 |
|------|------|
| 규정 다운로드 링크 사라짐 | 국가법령정보센터 검색, 식약처 공지사항 확인, 경고 기록 |
| PDF가 스캔 이미지 형식 | 문서 형식 명시, Data Engineer에 보고, OCR 필요성 검토 |
| 시행일 정보 누락 | 최대한 공식 자료에서 확인, 확인 불가 시 "확인 필요" 표시 |
| 문서 버전 갱신 발견 | 더 최신 버전 다운로드, 인벤토리에 버전 변경 기록 |

## 테스트 시나리오

### Happy Path
```
1. 6개 문서 모두 정상 다운로드
2. 각 문서의 공식 시행일, 최신 개정일 기록 완료
3. _workspace/REGULATIONS_INVENTORY.md 생성
4. 모든 PDF가 _workspace/regulations/ 저장
5. Data Engineer의 청킹 준비 완료
```

### 엣지 케이스
```
1. 식약처 링크 이동: 이전 URL과 새 URL 모두 기록
2. 문서 버전 여러 개: 최신 버전만 수집, 변경 내역 주석으로 기록
3. 협회 자료 비공개: 대체 자료 또는 공개 가이드 활용
```
