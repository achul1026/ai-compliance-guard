# 🛡️ AI 기반 자동 컴플라이언스(준수성) 검사기 개발 계획서
> **AI-Powered Automated Compliance Auditor & Remediation System**

본 문서는 복잡한 규정, 법률 매뉴얼, 내부 가이드라인을 기반으로 입력된 텍스트(계약서, 광고 카피 등)의 위반 소지를 실시간으로 검사하고 안전한 대안을 제시하는 **AI Micro-SaaS** 프로젝트의 기술 아키텍처 및 개발 로드맵을 정리한 파일입니다.

---

## 1. 프로젝트 개요
* **목적**: 복잡한 법적 리스크(광고 심의, 독소조항 등)를 사전 예방하여 기업의 법무 비용을 절감하고, 고도의 AI 아키텍처 구현을 통한 기술 스택 업그레이드.
* **핵심 가치**: 
    * **환각(Hallucination) 제로화**: 법률 문서 특성에 맞춘 정밀 검색 및 교차 검증.
    * **Remediation(대안 제시)**: 단순 위험 탐지를 넘어 바로 사용할 수 있는 안전한 대체 문구 제안.
    * **Data Privacy**: 기업 고객이 안심하고 쓸 수 있는 보안 파이프라인 구축.

---

## 2. 시스템 아키텍처 (System Architecture)

시스템은 유저가 입력한 컨텐츠를 분석하기 위해 **Hybrid RAG**로 근거 법령을 검색한 후, **Multi-Agent** 파이프라인을 거쳐 최종 리포트를 생성하는 구조를 가집니다.

```
[유저 입력: 광고 카피 / 계약서] 
       │
       ▼
 ┌───────────┐      1. 규정 검색 (Hybrid RAG)      ┌──────────────────┐
 │  Orche스  │ ──────────────────────────────────> │ 벡터 DB + 키워드  │
 │  트레이터 │ <────────────────────────────────── │ (법령/심의 가이드)│
 └───────────┘          관련 조항/판례 컨텍스트      └──────────────────┘
       │
       ├─► [에이전트 A: 검사관 (Auditor)] ──► 법적 위반 소지 및 리스크 리포트 작성
       │                                         │
       │                                         ▼
       ├─► [에이전트 B: 비판관 (Critique)] ──► 근거 법령이 맞는지 교차 검증 (환각 차단)
       │                                         │
       │                                         ▼
       └─► [에이전트 C: 교정관 (Remediator)] ─► 안전한 대체 문구/독소조항 수정안 제안
                                                 │
                                                 ▼
                                    [최종 컴플라이언스 리포트 출력]
```

### 💡 핵심 AI 파이프라인 특징
1.  **Stateful Workflow (상태 기반 워크플로우)**: 검사 → 비판 → 수정 단계가 순차적으로 진행되며, 비판 에이전트가 환각이나 오류를 감지하면 다시 검사 단계로 피드백을 보내는 순환 구조(Loop) 제어.
2.  **Hybrid RAG**: 법률 조항 번호나 고유 명사 매칭을 위한 **키워드 검색(BM25)**과 문맥적 의미를 찾는 **벡터 검색(Semantic)**을 결합하고, 관련성 높은 문서 순으로 재정렬하는 **Re-ranking** 프로세스 적용.

## 3. 추천 기술 스택 (Tech Stack)

| 레이어 | 기술 스택 | 선정 이유 |
| :--- | :--- | :--- |
| **Backend** | Java / Spring Boot (Gradle 멀티모듈) | 엔터프라이즈급 안정성, 복잡한 비즈니스 로직 및 확장성 확보 |
| **AI Orchestration** | LangChain4j / LangGraph | 멀티 에이전트의 복잡한 상태(State) 및 루프 워크플로우 정밀 제어 |
| **Database** | PostgreSQL + pgvector | 관계형 데이터(유저, 이력)와 벡터 데이터를 하나의 인프라로 통합 관리 |
| **Search Engine** | Hybrid RAG (BM25 + Vector) | 법률 도메인 특유의 고유 명사 및 조항 번호 검색 정확도 극대화 |
| **LLM Engine** | OpenAI GPT-4o / Upstage Solar | 복잡한 법리적 추론 능력 및 한국어 법률/규정 특화 모델 활용 |
| **Security** | JWT (HttpOnly Cookie), AES-256 | 사용자 분석 이력 및 세션의 안전한 보호 및 프라이버시 준수 |

## 4. 3단계 개발 로드맵 (Development Roadmap)

### 📅 1단계: 타겟 도메인 압축 및 Hybrid RAG 구축 (1~2개월)
* **범위 제한**: `스타트업/소상공인 쇼핑몰 이용약관 독소조항 검사` 또는 `건강기능식품/뷰티 SNS 광고 심의 가이드 검사` 중 1개의 틈새 도메인을 타겟으로 설정.
* **데이터 구축**: 타겟 도메인의 최신 규정 매뉴얼 및 가이드라인 PDF 파싱 및 청킹(Chunking) 최적화.
* **검색 파이프라인**: PostgreSQL + pgvector 기반으로 하이브리드 검색을 구현하여 질문에 맞는 정확한 조항을 찾아오는 백엔드 API 완성.

### 📅 2단계: 멀티 에이전트 파이프라인 및 UI 구현 (1개월)
* **에이전트 오케스트레이션**: LangGraph 혹은 LangChain4j를 활용하여 `검사관(Audit) - 비판관(Critique) - 교정관(Remediate)` 구조의 파이프라인 연결.
* **프론트엔드 연동**: 유저가 문서를 업로드하거나 텍스트를 입력하면 위험 구절에 **하이라이팅(위험도별 색상 차등)**을 표기하고 대안 툴팁을 보여주는 대시보드 UI 구현.

### 📅 3단계: 엔터프라이즈 보안 적용 및 수익화 런칭 (1개월)
* **데이터 프라이버시(보안) 강화**: 외부 LLM API 호출 시 사용자 데이터가 AI 학습에 활용되지 않도록 옵션 처리(`data_collection: false`), 입력 데이터 마스킹 기술 검토.
* **PG 연동 및 요금제**: 토스페이먼츠 등 결제 모듈 연동.
    * *Free Tier*: 월 3~5회 무료 검사 제공.
    * *Paid Tier*: 건당 크레딧 차감 방식 또는 소상공인/스타트업 대상 월간 정기 구독(SaaS) 모델 도입.
